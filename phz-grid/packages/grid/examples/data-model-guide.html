Dashboard Data Model

A 5-layer computation pipeline that transforms raw data fields into actionable KPIs. Each layer builds on the previous one, creating a directed acyclic graph (DAG) of dependencies.

Data flows left to right. Each layer can reference layers to its left. For example, a Metric can reference Fields, Parameters, and Calculated Fields — but not KPIs.

Fields

Layer 1
📄
What Are Fields?
Auto-detected from your data

Fields are the raw columns in your dataset. They are automatically detected when you load data into the dashboard — you don't create or modify them manually.

Each field has a name and a type inferred from the first data row:

  • string — text values
  • number — numeric values
  • boolean — true / false
  • date — date/time values
🏷️
Reference Syntax
[field_name]

Reference fields in expressions using square brackets:

// Access a field value in an expression [salary] * 1.1 [first_name] + " " + [last_name] IF([status] eq "Active", 1, 0)
Fields are read-only. To create derived values, use Calculated Fields (layer 3).

Parameters

Layer 2
⚙️
What Are Parameters?
User-adjustable inputs

Parameters are adjustable inputs that control dashboard behavior — targets, thresholds, weights. Users can change these without modifying the data model.

When to use:

  • Dashboard filters and user preferences
  • Adjustable targets (e.g. performance goal = 85%)
  • Dynamic thresholds for KPI status bands
  • Weights for scoring formulas

Parameter types:

  • number — with optional min/max/step
  • string — free text
  • date — date picker
  • boolean — true/false toggle
  • select — dropdown with predefined options
📝
Reference Syntax
$param_name

Reference parameters in expressions using the dollar sign:

// Use a parameter in a calculated field [salary] * $bonus_rate // Compare against a parameter threshold IF([score] gte $target, "Pass", "Fail")

Programmatic API:

const store = createDashboardDataModelStore(fields); store.addParameter({ id: parameterId('bonus_rate'), name: 'Bonus Rate', type: 'number', defaultValue: 0.1, min: 0, max: 1, step: 0.05, });

Calculated Fields

Layer 3
🧮
What Are Calculated Fields?
Per-row transformations

Calculated fields create new columns from existing data. They run on each row before any aggregation — perfect for transformations, ratios, and categorizations.

When to use:

  • Combining fields: [first_name] + " " + [last_name]
  • Normalizing: [rating] / [max_rating] * 100
  • Categorizing: IF([salary] > 100000, "High", "Normal")
  • Parameterized: [salary] * $bonus_rate
Calculated Fields vs Metrics: Calculated fields run per-row BEFORE aggregation. Metrics aggregate AFTER. If you need a per-row value, use a calculated field. If you need a single aggregate (sum, average), use a metric.
📝
Reference Syntax
~calc_name

Reference calculated fields in other calculated fields using the tilde:

// Use a calculated field in another ~performance_score / 100

Example expressions:

// Salary with bonus [salary] * (1 + $bonus_rate) // Performance percentage [rating] / [max_rating] * 100 // Active flag IF([status] eq "Active", 1, 0) // Clamped score CLAMP([score], 0, 100)

Output types: Number, Text, Boolean, Date

Metrics

Layer 4
📊
What Are Metrics?
Aggregate values

Metrics aggregate data across all rows into a single value (sum, average, count, etc.). They come in two modes:

Simple metrics: Single field + aggregation

  • avg([salary]) — average salary
  • count([id]) — headcount
  • sum([revenue]) — total revenue
  • min([age]) / max([age])

Composite metrics: Expressions combining other metrics

  • @total_revenue / @headcount — revenue per employee
  • @active_count / @total_count * 100 — active percentage

7 aggregations: sum, avg, count, min, max, first, last

📝
Reference Syntax
@metric_name

Reference metrics in composite metric expressions using the at sign:

// Revenue per employee @total_revenue / @headcount // Active rate percentage @active_employees / @total_employees * 100

Programmatic API:

engine.metrics.register({ id: metricId('avg_salary'), name: 'Average Salary', dataProductId: 'employees', formula: { type: 'simple', field: 'salary', aggregation: 'avg', }, format: { type: 'currency', decimals: 0, prefix: '$' }, });

KPIs

Layer 5
🎯
What Are KPIs?
Tracked business outcomes

KPIs track key business outcomes with targets and status thresholds. Link a KPI to a metric to automatically compute its value from data.

Key properties:

  • Metric link — which metric provides the value
  • Target — the goal value (e.g. 90%)
  • Direction — Higher is Better or Lower is Better
  • Status — automatically computed: OK / Warning / Critical
🚦
Threshold Modes
Simple & Custom Bands

Simple thresholds: Two values that divide into 3 zones (OK / Warning / Critical).

thresholds: { ok: 90, warn: 70 } // ≥ 90 = OK, 70-89 = Warning, < 70 = Critical

Custom Bands: Define 2-5 colored status zones with dynamic threshold sources.

bands: [ { label: 'Critical', color: '#DC2626', upTo: { type: 'static', value: 70 } }, { label: 'Warning', color: '#D97706', upTo: { type: 'parameter', parameterId: 'warn_threshold' } }, { label: 'On Track', color: '#16A34A', upTo: { type: 'static', value: 100 } }, ]
Band thresholds can be dynamic — sourced from parameters or metrics, not just static values.

Quick Start

Load Data

Upload a CSV or pass data programmatically. Fields are auto-detected from column names and types.

const engine = createBIEngine({ ... }); const data = [ { name: 'Alice', salary: 95000, rating: 4.2, department: 'Engineering' }, { name: 'Bob', salary: 82000, rating: 3.8, department: 'Marketing' }, ... ];

Create a Parameter

Add adjustable inputs for dashboard consumers. Here we create a target percentage parameter.

store.addParameter({ id: parameterId('target_pct'), name: 'Target %', type: 'number', defaultValue: 85, min: 0, max: 100, step: 5, });

Create a Calculated Field

Derive a new per-row column. This creates a normalized score (0-100) from raw rating.

store.addCalculatedField({ id: calculatedFieldId('norm_rating'), name: 'Normalized Rating', outputType: 'number', expression: { /* AST for: [rating] / 5 * 100 */ }, });

Create a Metric

Aggregate across all rows. Here we compute the average normalized rating.

engine.metrics.register({ id: metricId('avg_rating'), name: 'Avg Rating', dataProductId: 'employees', formula: { type: 'simple', field: 'norm_rating', aggregation: 'avg' }, format: { type: 'number', decimals: 1, suffix: '%' }, });

Create a KPI

Track the metric against a target, with automatic status (OK / Warning / Critical).

engine.kpis.register({ id: kpiId('team-performance'), name: 'Team Performance', metricId: metricId('avg_rating'), target: 85, direction: 'higher_is_better', thresholds: { ok: 85, warn: 70 }, });
All of these can also be created visually in the Dashboard Studio using the data model sidebar. Click the + button next to any layer section to open the guided creation modal.
